# Ford MPV and SUV Security Split Audit

**Version:** 1.0  
**Status:** In progress  
**Last updated:** 2026-07-14  
**Scope:** UK-market Ford MPV/SUV badge records, security-generation boundaries only

## Purpose

This document records provisional record boundaries for Ford MPV and SUV models before key, transponder, tool and FDRS data are approved.

A facelift is not treated as a security split unless locksmith-relevant evidence proves a change.

## Galaxy and S-Max

### Confirmed generation boundaries

- Galaxy 1995–2006 is the Volkswagen Sharan/SEAT Alhambra joint-venture family and must not inherit Ford PATS assumptions.
- Galaxy 2006–2015 is the first Ford-developed Galaxy generation with no Volkswagen Group platform content.
- Galaxy 2015–2023 is the later Ford generation.
- S-Max 2006–2015 is the first S-Max generation.
- S-Max 2015–2023 is the second generation.
- Production of Galaxy and S-Max ended in April 2023.

### Provisional record plan

| Proposed record | Status | Reason |
|---|---|---|
| Galaxy 1995–2000 | shared_platform | Early VAG joint-venture generation; verify immobiliser generation. |
| Galaxy 2000–2006 | shared_platform | Facelifted VAG joint-venture generation; verify whether immobiliser generation changed. |
| Galaxy 2006–2015 keyed | split_required | Ford-developed generation; verify keyed continuity. |
| Galaxy 2006–2015 passive start | split_required | Separate where passive start was fitted. |
| Galaxy 2015–2023 keyed | split_required | Later Ford generation; verify keyed availability. |
| Galaxy 2015–2023 passive start | split_required | Separate smart-key record; verify online/FDRS requirements. |
| S-Max 2006–2015 keyed | split_required | First generation; verify keyed continuity. |
| S-Max 2006–2015 passive start | split_required | Separate where passive start was fitted. |
| S-Max 2015–2023 keyed | split_required | Second generation; verify keyed availability. |
| S-Max 2015–2023 passive start | split_required | Separate smart-key record; verify online/FDRS requirements. |

## C-Max and Grand C-Max

### Confirmed generation boundaries

- Focus C-Max was sold as a distinct badge from 2003 to 2007.
- C-Max 2007–2010 is the renamed/facelifted first-generation family.
- The second-generation C-Max and Grand C-Max launched for 2010 and ran to 2019.
- The 2015 facelift remains inside the second generation unless security evidence proves a change.

### Provisional record plan

| Proposed record | Status | Reason |
|---|---|---|
| Focus C-Max 2003–2007 keyed | verification_required | Badge-specific record; verify one security generation. |
| C-Max 2007–2010 keyed | verification_required | Renamed first-generation family; verify security continuity. |
| C-Max 2010–2019 keyed | split_required | Second generation; do not split at 2015 facelift without evidence. |
| C-Max 2010–2019 passive start | split_required | Separate where passive start was fitted. |
| Grand C-Max 2010–2019 keyed | split_required | Separate badge record; verify one keyed security generation. |
| Grand C-Max 2010–2019 passive start | split_required | Separate where passive start was fitted. |

## B-Max

### Confirmed generation boundary

- One B-Max generation was produced from 2012 to 2017 on the Ford global B-car/Fiesta-derived platform.
- There was no normal mid-cycle facelift requiring a new model-generation record.

|---|---|---|

## Evidence Quality

The generation boundaries above are suitable for inventory correction. They do not approve:

- blade or emergency blade;
- transponder;
- frequency;
- Add Key/AKL;
- tool support;
- exact passive-start trim/build availability;
- FDRS, online or gateway requirements.

## Master-File Actions

1. Keep Galaxy 1995–2006 under Ford but assign VAG-derived security research.
2. Keep Galaxy 2006–2015 and 2015–2023 as separate Ford generations.
3. Keep S-Max 2006–2015 and 2015–2023 as separate generations.
4. Keep C-Max/Grand C-Max 2010–2019 together across the 2015 facelift unless security evidence proves otherwise.
5. Keep B-Max 2012–2017 as one generation.
6. Create separate keyed/passive records only where fitment is confirmed and programming data differs.
